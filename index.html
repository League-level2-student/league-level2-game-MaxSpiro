Cactus Jump

by: Max Spiridonov

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I made this game because an infinite runner game is not too difficult. Being a somewhat beginner programmer, I felt that this would be challenging enough for me.

2. What I Learned

Some of my classmates thought that the game was too easy, and others thought that it was too hard, so I decided to make a difficulty selection screen where the user can choose how hard the game is.

3. Architecture of My Game

I used several classes in order to better structure my game. I had a GamePanel class where most of the graphics went, and I also put all of the methods that dealt with the 'stages' in it. This is the main class where I had objects of my other classes. The ObjectManager class organized all of my other objects such as the arraylist of cactuses and the dinosaur class. This made it easier to update the classes.

4. Technical Implementation

One thing I am proud of in this game is the jump method I used for the dinosaur. I used a variable called weight in order to determine how fast the dinosaur falls, so the higher the number, the faster it falls. I found this to be one of the most challenging parts of the project.

5. Most Challenging Part

The most challenging part of the project was the bug fixing that I had to do. For a while, the weight variable was not working, so the dinosaur would just float upwards when it jumped. For around two weeks, I did not know what was causing it, and I had to just keep testing different parts of the code to see if it was the problem. It became frustrating, for I felt helpless, but I persevered through it and eventually found the problem.

6. Future Fixes and Updates

I think that I would like to make some of my own graphics for the background, dinosaur, and cactus. Also, I would add more enticing features such as power-ups in order to make it more interesting.
